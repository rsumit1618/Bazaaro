# Firebase, GitHub Pages, and Offline Guide

## Can The Web App Open From A GitHub Link?

Yes. Flutter web builds static files, so the customer web app can be hosted on GitHub Pages.

If the GitHub Pages URL is:

```text
https://<username>.github.io/bazaaro/
```

Build with:

```bash
cd apps/customer_app
flutter build web --release --base-href /bazaaro/
```

Then publish:

```text
apps/customer_app/build/web
```

to GitHub Pages.

## GitHub Pages Limitation

GitHub Pages only hosts static files. It does not run a backend. This is fine for Bazaaro because:

- Flutter web is static after build.
- Firebase Realtime Database is remote.
- Local seed data can load from app assets.

## Firebase Hosting Alternative

Firebase Hosting is better when using Firebase because it supports clean hosting rewrites and easy deploys:

```bash
cd apps/customer_app
flutter build web --release
cd ../..
firebase deploy --only hosting
```

The repo uses:

```text
firebase.json
```

with SPA rewrite to `index.html`.

## Offline Handling Strategy

Bazaaro should not show a blank screen when the network is down.

Current strategy:

- Local JSON seed data provides initial catalog data.
- Local SQLite/cache layer supports catalog persistence.
- Firebase initialization is non-blocking.
- Repository provider falls back to local data when Firebase is unavailable.
- UI has loading, empty, and error states through App Stream Kit.

Recommended production strategy:

- Cache catalog, categories, banners, and recently viewed products locally.
- Queue cart/order mutations when possible.
- Show offline banners on mobile and web.
- Retry Firebase sync when the connection returns.
- Avoid blocking app startup on Firebase.

## Mobile Offline

Mobile can use:

- SQLite for catalog/cache.
- Shared preferences or secure storage for small session flags.
- Cached network images where package support is available.
- A connectivity provider to show offline status.

Expected behavior:

- Home loads from local cache/seed.
- Search works on cached products.
- Product detail works for cached products.
- Checkout requires login and network for final order sync.
- If offline during checkout, show a clear retry message.

## Web Offline

Flutter web can use:

- Service worker/PWA cache generated by Flutter web.
- Browser cache for static assets.
- Local seed JSON bundled with the app.
- Firebase SDK reconnect behavior.

Expected behavior:

- App shell opens if it has been loaded before.
- Static assets can be cached.
- Local seed catalog can render.
- Firebase-only actions show friendly offline errors.

## About Base64 Images

Storing large base64 images in Realtime Database is not recommended for production because it increases document size, bandwidth, parsing time, and database cost.

Better options:

- Use Firebase Storage when billing/configuration is acceptable.
- Use a CDN or image hosting service.
- Use optimized remote image URLs in seed/admin data.
- Use small base64 placeholders only for tiny thumbnails.
- If base64 is unavoidable, split into chunks and reconstruct only when needed, but keep product listing images small.

For the current demo, using image URLs/local assets is cleaner and faster.
